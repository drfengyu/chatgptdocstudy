开发人员快速入门

启动并运行 OpenAI API​

寻找 ChatGPT？前往 chat.openai.com。

OpenAI API 为开发人员提供了一个简单的界面，可以在其应用程序中创建智能层，由 OpenAI 最先进的模型提供支持。Chat Completions 端点为 ChatGPT 提供支持，并提供了一种将文本作为输入并使用 GPT-4 等模型生成输出的简单方法。

想直接跳转到代码吗？

跳过快速入门，深入了解 API 参考。

本快速入门旨在帮助设置本地开发环境并发送第一个 API 请求。如果您是一位经验丰富的开发人员，或者只想深入了解使用 OpenAI API，那么 GPT 指南的 API 参考是一个很好的起点。在本快速入门中，你将了解：

  • 如何设置开发环境
  • 如何安装最新的 SDK
  • OpenAI API 的一些基本概念
  • 如何发送您的第一个 API 请求

如果您遇到任何挑战或对入门有疑问，请加入我们的开发者论坛。

帐户设置​

首先，创建一个 OpenAI 帐户或登录。接下来，导航到 API 密钥页面和“创建新的密钥”，可以选择命名密钥。确保将其保存在安全的地方，不要与任何人分享。

快速入门语言选择​

选择要开始使用 OpenAI API 的工具或语言。

Python 是一种流行的编程语言，由于其易用性，通常用于数据应用程序、Web 开发和许多其他编程任务。OpenAI 提供了一个自定义 Python 库，使在 Python 中使用 OpenAI API 变得简单高效。

第 1 步：设置 Python​

安装 Python​

要使用 OpenAI Python 库，您需要确保已安装 Python。有些计算机预装了 Python，而其他计算机则需要您自己设置。若要测试是否安装了 Python，可以导航到“终端”或“命令行”：

  • MacOS：打开终端：您可以在“应用程序”文件夹中找到它，也可以使用 Spotlight（Command + 空格键）搜索它。
  • Windows：打开命令提示符：您可以通过在开始菜单中搜索“cmd”来找到它。 接下来，输入单词，然后按 Return/Enter。如果您进入 Python 解释器，则您的计算机上已经安装了 Python，您可以转到下一步。如果您收到类似“错误：找不到命令 python”的错误消息，您可能需要安装 Python 并在终端/命令行中使其可用。python

要下载 Python，请前往 Python 官方网站并下载最新版本。要使用 OpenAI Python 库，您至少需要 Python 3.7.1 或更高版本。如果您是第一次安装 Python，您可以按照官方的 Python 初学者安装指南进行操作。

设置虚拟环境（可选）​

安装 Python 后，最好创建一个虚拟 python 环境来安装 OpenAI Python 库。虚拟环境为要安装的 Python 包提供了一个干净的工作空间，这样您就不会与为其他项目安装的其他库发生冲突。您不需要使用虚拟环境，因此如果您不想设置虚拟环境，请跳到步骤 3。

为了创建虚拟环境，Python 提供了一个内置的 venv 模块，该模块提供了虚拟环境所需的基本功能。运行以下命令将在您在终端/命令行中选择的当前文件夹中创建一个名为“openai-env”的虚拟环境：

python -m venv openai-env

创建虚拟环境后，您需要激活它。在 Windows 上，运行：

openai-env\Scripts\activate

在 Unix 或 MacOS 上，运行：

source openai-env/bin/activate

激活虚拟环境后，您应该会看到终端/命令行界面略有变化，它现在应该在光标输入部分的左侧显示“openai-env”。有关使用虚拟环境的更多详细信息，请参阅官方 Python 文档。

安装 OpenAI Python 库 安装 Python 3.7.1 或更高版本并（可选）设置虚拟环境后，可以安装 OpenAI Python 库。从终端/命令行，运行：

pip install --upgrade openai

完成此操作后，运行将显示已在当前环境中安装的 Python 库，这应该确认 OpenAI Python 库已成功安装。pip list

第 2 步：设置 API 密钥​

为所有项目设置 API 密钥（推荐）​

使您的 API 密钥可供所有项目访问的主要优点是 Python 库将自动检测并使用它，而无需编写任何代码。

MacOS操作系统​

  1. 打开终端：您可以在“应用程序”文件夹中找到它，也可以使用 Spotlight（Command + 空格键）搜索它。

  2. 编辑 Bash 配置文件：使用命令 或（对于较新的 MacOS 版本）在文本编辑器中打开配置文件。nano ~/.bash_profilenano ~/.zshrc

  3. 添加环境变量：在编辑器中，添加以下行，替换为您的实际 API 密钥：your-api-key-here

export OPENAI_API_KEY='your-api-key-here'

  1. 保存并退出：按 Ctrl+O 写入更改，然后按 Ctrl+X 关闭编辑器。

  2. 加载配置文件：使用命令 或加载更新的配置文件。source ~/.bash_profilesource ~/.zshrc

  3. 验证：通过在终端中键入来验证设置。它应该显示您的 API 密钥。echo $OPENAI_API_KEY

窗户​

  1. 打开命令提示符：您可以通过在开始菜单中搜索“cmd”来找到它。

  2. 在当前会话中设置环境变量：要在当前会话中设置环境变量，请使用以下命令，并替换为您的实际 API 密钥：your-api-key-here

setx OPENAI_API_KEY "your-api-key-here"

此命令将为当前会话设置OPENAI_API_KEY环境变量。

  1. 永久设置：要使设置永久化，请通过系统属性添加变量，如下所示：
  • 右键单击“这台电脑”或“我的电脑”，然后选择“属性”。
  • 点击“高级系统设置”。
  • 单击“环境变量”按钮。
  • 在“系统变量”部分，点击“新建...”并输入 OPENAI_API_KEY 作为变量名称，输入您的 API 密钥作为变量值。

验证：若要验证设置，请重新打开命令提示符并键入以下命令。它应该显示你的 API 密钥：echo %OPENAI_API_KEY%

  1. 为单个项目设置 API 密钥 如果您只想让单个项目访问您的 API 密钥，您可以创建一个包含 API 密钥的本地文件，然后将该 API 密钥与后续步骤中所示的 Python 代码显式使用。.env

首先转到要在其中创建文件的项目文件夹。.env

为了使版本控制忽略 .env 文件，请在项目目录的根目录中创建一个 .gitignore 文件。添加一行带有 .env 的行，以确保您的 API 密钥或其他机密不会通过版本控制意外共享。 使用终端或集成开发环境 （IDE） 创建 and 文件后，复制您的密钥并将其设置为文件中的 。如果您尚未创建密钥，可以在 API 密钥页面上进行创建。.gitignore.envOPENAI_API_KEY.env

该文件应如下所示：.env

# Once you add your API key below, make sure to not share it with anyone! The API key should remain private.
OPENAI_API_KEY=abc123

可以通过运行以下代码来导入 API 密钥：

from openai import OpenAI

client = OpenAI()
# defaults to getting the key using os.environ.get("OPENAI_API_KEY")
# if you saved the key under a different environment variable name, you can do something like:
# client = OpenAI(
#   api_key=os.environ.get("CUSTOM_ENV_NAME"),
# )

第 3 步：发送您的第一个 API 请求 发出 API 请求 配置并设置 Python 密钥后，最后一步是使用 Python 库向 OpenAI API 发送请求。为此，请创建一个使用 th 终端或 IDE 命名的文件。openai-test.py

在文件中，复制并粘贴以下示例之一：

聊天完成

from openai import OpenAI
client = OpenAI()

completion = client.chat.completions.create(
  model="gpt-3.5-turbo",
  messages=[
    {"role": "system", "content": "You are a poetic assistant, skilled in explaining complex programming concepts with creative flair."},
    {"role": "user", "content": "Compose a poem that explains the concept of recursion in programming."}
  ]
)

print(completion.choices[0].message)

要运行代码，请输入终端/命令行。python openai-test.py

聊天完成示例仅突出了我们模型的一个优势领域：创造力。在一首格式良好的诗歌中解释递归（编程主题）是最好的开发人员和最好的诗人都会努力解决的问题。在这种情况下，毫不费力地做到了。gpt-3.5-turbo

后续步骤 现在您已经发出了第一个 OpenAI API 请求，是时候探索其他可能性了：

  • 有关我们的模型和 API 的更多详细信息，请参阅我们的 GPT 指南。
  • 访问 OpenAI Cookbook，了解深入的示例 API 用例，以及常见任务的代码片段。
  • 想知道 OpenAI 的模型有什么能力吗？查看我们的示例提示库。
  • 想在不编写任何代码的情况下试用 API？开始在 Playground 中进行试验。
  • 在开始构建时，请牢记我们的使用政策。
